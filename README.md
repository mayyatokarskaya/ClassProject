# Проект: Задание 1

## Описание проекта

Этот проект представляет собой реализацию двух классов: `Product` и `Category`. Классы предназначены для управления
товарами и категориями в рамках небольшой системы учета.

## Классы

### Класс `Product`

Класс `Product` представляет товар и содержит следующие атрибуты:

- **name** (str): Название товара.
- **description** (str): Описание товара.
- **price** (float): Цена товара.
- **quantity** (int): Количество товара в наличии.

### Класс `Category`

Класс `Category` представляет категорию товаров и содержит следующие атрибуты:

- **name** (str): Название категории.
- **description** (str): Описание категории.
- **products** (list): Список товаров, принадлежащих этой категории.

Кроме того, класс `Category` имеет два атрибута класса:

- **category_count** (int): Общее количество категорий.
- **product_count** (int): Общее количество товаров во всех категориях.

### Классы-наследники

Созданы два класса-наследника класса `Product`: **Смартфон** (`Smartphone`) и **Трава газонная** (`LawnGrass`).

#### Класс `Smartphone`:

Класс `Smartphone` расширяет функциональность `Product` и включает дополнительные атрибуты:

- **efficiency** (float): Производительность устройства.
- **model** (str): Модель смартфона.
- **memory** (int): Объем встроенной памяти.
- **color** (str): Цвет смартфона.

#### Класс `LawnGrass`:

Класс `LawnGrass` также расширяет `Product` и содержит дополнительные атрибуты:

- **country** (str): Страна-производитель.
- **germination_period** (str): Срок прорастания травы.
- **color** (str): Цвет травы.

## Ограничения

### Ограничения сложения

Доработана функциональность сложения таким образом, чтобы можно было складывать товары только из одинаковых классов
продуктов. Новая функциональность не позволяет сложить объекты разных классов, например, смартфон и траву газонную. При
попытке сложения таких экземпляров выбрасывается ошибка `TypeError`.

Для реализации ограничений используется функция `type()`.

### Ограничения добавления продукта

Доработан метод, который добавляет продукт в категорию, таким образом, чтобы не было возможности добавить вместо
продукта или его наследников любой другой объект. Для реализации этих ограничений используется функция `isinstance()`.

## Наследование

### Классы:

1. **BaseProduct** — базовый абстрактный класс для всех продуктов.
2. **Product** — класс продукта, который наследует `BaseProduct` и включает функциональность для работы с ценой,
   количеством, а также поддерживает магические методы для сложения объектов.
3. **PrintMixin** — класс-миксин, который добавляется в цепочку наследования классов продуктов, чтобы при создании
   нового объекта выводить информацию о нем в консоль.
4. **Smartphone** — класс, представляющий смартфон, наследуется от `Product`.
5. **LawnGrass** — класс, представляющий газонную траву, наследуется от `Product`.

### 1. **BaseProduct**

Это абстрактный класс, который служит родительским для всех классов продуктов. Он определяет обязательные методы, такие
как `new_product`, который используется для создания объектов из данных, переданных в виде словаря.
